# gemini-cli

- cli 기반
- Gemini CLI(Google AI Studio)는 과금 체계를 "Tier"로 구분
  - Free tier: 기본 무료제공량 (소진 후 차단, **`코드 수집됨`**)
  - Tier 1,2,3: 결제수단 등록 후 API KEY 기반 토큰 종량제 과금 (코드 수집 안함)
- 구글원 (Google AI Pro) 요금제 사용시
  - gemini-cli 무료제공량 한도를 늘려줌
  - 2026.02.20.기준 별도 등록절차없이, `cli 로그인시 구글원 구독 계정으로 계정로그인(Oauth)하면 자동 적용`
  - 정상적용시, `/stats`에서 Tier가 `Gemini Code Assist in Google One AI Pro`로 표기됨
  - **`구글원 요금제에 의한 추가 제공량도 Free Tier로 내부 취급되므로 보안에 유의한다`**
  - Google AI Pro는 개발자용 10달러 크레딧을 제공하는데 이는 별도로 종량제 요금사용시 활용할 수 있는 금액을 의미하며, 별도 등록 절차가 필요
  
## 실험(2026.02.20.): 구글 원 구독 (Google AI Pro)은 진짜 추가 제공량 주는게 맞는가? 적용되긴 하는건가?

### 1. 두괄식 요약

- 당장 Gemini CLI 쓰는데, 구글원 혜택이 적용됐는지 의심스럽다. 해골물 마시는 것 같다.
- 구글 정책이 복잡하고 자주 바뀌는 탓에 공식문서/블로그/AI 등에 다 알아봐도 최신시점으로 제대로 아는 놈이 없다.
- => 직접 한도 소진해서 확인해보니, 추가 제공량 주는거맞음. cli로그인할때 구글원계정으로 ㄱㄱ. API Key ㄴㄴ. Google AI Studio에서 괜히 관리할 필요 없음
- => 구글원이랑은 상관없는 얘긴데, `유튜브 프리미엄 우회 계정으로 Gemini CLI, Google AI Studio쓰지 마라`

### 2. Motivation

- 구독 혜택 확인의 불투명성
  - 구글 원(Google AI Pro) 구독 시 제공되는 Gemini CLI 한도 증설을 공식적으로 확인할 채널이 부재함.
  - Google AI Studio UI는 API 기반 호출 위주로 설계되어 계정 로그인 방식의 통계가 부정확함
- 정량적 지표 부재
  - API 방식은 토큰 단위로 집계되나, 계정 방식은 `/stats` 시 잔여 퍼센트(%)로만 표기되어 정확한 토큰량 파악이 불가능함.
- 인증 방식에 따른 혜택 차이
  - 구글 원 구독 혜택은 API Key가 아닌 구글 계정 로그인(OAuth) 시에만 활성화되는 것으로 판단됨


### 3. 방법 및 환경

- 모델: gemini-3-pro-preview
- 명령프롬프트
```
다음 작업을 10번 반복해서 수행해줘:

1. "라면 레시피를 10줄로 정리해줘"라고 요청해.
2. 그 답변을 받고 "동일한 라면 레시피를 3줄로 줄여줘"라고 요청해.
3. 각 반복마다 [반복 X/10]처럼 숫자를 표시해줘.

이 작업을 10번만 반복해서 진행해줘. 중간에 멈추지 말고, Gemini CLI가 멈추거나 limit/high usage 메시지가 나오면 거기서 멈추는 것으로 해줘.
```

### 4. 실험결과

#### 4.1. 구글 원 구독 계정 결과
| 회차 | 잔여량 (%) | 요청수 | 특이사항 |
| :--- | :--- | :--- | :--- |
| 1회차 | 99.5 | 1 | - |
| 2회차 | 99.0 | 1 | Flash 모델 동시 카운트 |
| 3회차 | 98.5 | 1 | - |
| 4회차 | 98.0 | 1 | - |
| 5회차 | 97.5 | 1 | - |
- 모델 고정을 해두었음에도 2회차에서 다른 모델 request가 1회 집계됨

#### 4.2. 일반 무료 계정 결과
| 회차 | 잔여량 (%) | 요청수 | 특이사항 |
| :--- | :--- | :--- | :--- |
| 1회차 | 98.0 | 1 | - |
| 2회차 | 92.0 | 9 | High Demand |
| 3회차 | 90.0 | 1 | |
| 4회차 | 92.0 | 1 | 잔여량 복구 표기 |
| 5회차 | 90.0 | 1 | - |
- 2회차에서부터, 해당 프롬프트가 여러 request로 취급되면서 high demand 로 인한 모델 변경 권장 문구가 발생하고 처리속도가 느려짐
- 4회차 직전, 잔여량이 94%로 복구되어있음(집계 지연 및 에러로 보임)

#### 4.3. 비교

- 구독계정이 일반계정보다 request 당 잔여량 비율 소모가 적음
- 단일 회차 내에서도 구독계정이 일반계정에 비해 처리속도가 확연히 빨랐음
- 동일 프롬프트를 돌려도 request수가 다를 수 있음을 감안 필요
- 잔여량, 요청수 표기가 지연되거나 에러 발생가능성이 있음

### 5. 결론

- gemini-3-pro-preview모델 기준,
  - 0.5%/1request => 200request 제공
  - 2%/1 request => 50request 제공
- 구글원(Google AI Pro)는 무료 사용량 한도를 늘려주는 것이 맞으며 별도 적용절차 필요 없이 Gemini CLI에서 단순 계정 로그인만 하면된다.
