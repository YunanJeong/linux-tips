# Gemini CLI 프로젝트 설정 가이드

프로젝트 보안 및 협업 효율을 위해 아래 파일들을 프로젝트 루트(`.git`이 있는 곳)로 복사하여 사용하십시오.

### 1. 설정 파일 (Hierarchy)
- `.gemini/settings.json`: 이 프로젝트 내에서만 적용될 보안 및 도구 동작을 제어합니다.
- `.geminiignore`: 제가 읽어서는 안 되는 비밀번호, 개인키, 대용량 파일을 지정합니다. (보안 필수!)
- `GEMINI.md`: 프로젝트의 기술 스택, 아키텍처 가이드, 코딩 규칙을 담습니다.

### 2. 설정 경로 안내 (Global vs Local)
Gemini CLI는 계층형 설정을 지원하므로 상황에 맞는 경로에 파일을 위치시킵니다.

- 홈 디렉토리 (`~/.gemini/`): 전역(Global) 설정입니다. 모든 프로젝트에서 공통으로 지킬 개인적인 보안 규칙이나 통계 설정을 둡니다.
- 프로젝트 루트 (`<project-root>/.gemini/`): 로컬(Local) 설정입니다. 팀원들과 공유할 프로젝트 전용 기술 스택, 명령어 허용 목록, 코딩 컨벤션(GEMINI.md)을 둡니다. 
  - 로컬 설정이 전역 설정보다 항상 높은 우선순위를 가집니다.

### 3. 사용 방법
1. `.gemini/` 폴더와 `.geminiignore`, `GEMINI.md`를 프로젝트 루트에 복사합니다.
2. 각 파일의 내용을 프로젝트 성격에 맞춰 수정합니다.
3. 이제 이 폴더에서 Gemini CLI를 실행하면 설정된 규칙과 보안 정책이 즉시 적용됩니다.
